Security Mode (set at installation):
With the No Security model, all users have full access to all Dashboards. In a sense, all users have Admin rights to them. Anyone can create a Dashboard, and all Dashboards can be viewed and editted by all users. This model is ideal for standalone installations (where there is only one user) or small teams with a high level of trust.
Note that while editing a Dashboard, changes are never applied to the original. A separate Draft copy is created. This copy is visible only to the editor, while all other users will see only the original. The editor can use checkpoints (see File -> Checkpoints menu item) to store interim copies. The editor may role back to a previously stored checkpoint at any time. When the Draft copy is created, the system automatically creates the first checkpoint called 'Original'. The use of checkpoints is optional.
During editing, all changes are saved automatically to the Draft copy without any input from the editor. Once changes have been finalised and the editor wants to make these available to other users, he clicks the Save button. The Save button will replace the original copy with the Draft copy, and remove the Draft copy (and checkpoints).
Also, only one user at a time may edit a Dashboard. Once changes have started, the editor can Discard these changes (which will simply delete the Draft copy and all checkpoints). In order to allocate rights to individual users or groups, the Team security model must be used.

Your security mode (set by Admin):

There are 2 security modes, the default is Normal
In Strict mode, there are different User roles:
1. Admin - can Open, Edit, Request-QA, QA and Save all Dashboards
2. DashboardBuilder, which is a type of group membership, can perform the following actions on all Dashboards that the group has access to: a. Open (readonly) b. Edit c. Discard d. Request-QA e. Save
3. All other Users can perform all actions to any Dashboard that the user has access to, BUT does not belong to a group.
